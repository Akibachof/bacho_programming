勉強日記 2021/08/08

Macでやったことメモ

目標

自分のgithub

からターミナルに
git clone https://github.com/Akibachof/bacho_english.git
と
git clone https://github.com/Akibachof/bacho_programming.git
のコマンドを使って、web公開したサイトのリモートリポジトリとローカルを連携することで、atomからサイトを編集したい。


上記のコマンドをターミナルに入力すると、エラーが出た。

"xcrun: error: invalid active developer path"
そのため、 https://qiita.com/kooohei/items/6aada548eeb785dd2c91 を参考にコマンドラインデベロッパーツールとやらをDLした。
→ 解決


もう一度 git cloneを試みると、今度はリポジトリの複製がうまくいった。
しかし、新たなエラーが発生した。

fatal: Unable to find remote helper for 'https'
現状の問題を整理すると、

問題点1：gitcloneでリモートからコピーしてきたディレクトリがatom上で全然反応してくれない(pullとかpushのボタンが出ない)
問題点2：すでにgithubと連携してあったTCADフォルダもプロジェクトを開いても反応してくれない

どうやらhttpsのためのhelperがないかららしい。gitのバージョンが古いのが原因っぽい？(2番目の参考サイトより)
参考：

https://qiita.com/qualitia_cdev/items/8315e6d7badbb5ddd0c6

https://nplll.com/2019/10/macos-update-in-trouble/


そのままgitをアプデしてはつまらないので、インストールされてるgitのバージョンと最新バージョンを確認してみる。

git --versionで確認すると、2.30.1が入ってる模様。最新版は https://git-scm.com/を参照。
2021/8/8現在、ver 2.32.0が最新の模様


ということで、gitのアプデをするのにhomebrewを使うことにしたが、なぜかM1 Macには入ってなかった。
https://www.teamxeppet.com/macbook-m1-homebrew-install/ を参考にインスコ。

homebrewのインストールが済んだ。次にパスを通す必要があるらしい。
以下の2つ
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> /Users/"ユーザー名"/.zprofile
eval "$(/opt/homebrew/bin/brew shellenv)"
を打ちこめっていうのでその通りにした。brew helpコマンドを打つとちゃんとhelpが出てきたので、上手く行った模様。

brew -v と打つとバージョン3.2.6が無事インストールされたことがわかった。

ということでgitのアプデを行う。
brew install git
その場でバージョンを確認すると、2.30.1のまま。うまく行ってないのかと思いきや、 ターミナルのウィンドウを開いたままだと新しいgitのパスが変わらないことが原因らしい。
ターミナルを再起動してもう一回確認すると、2.32.0が入ってることが確認できた。

さいごに一言

そもそもgitのアップデートするなら
brew upgrade git
でええやんけ

上手にできました
