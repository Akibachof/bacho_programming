Linux系のターミナル使い方

  1. 個人アカウントでのsuの入り方

    2. su → return → rootのパスワードを入力


  2. ジョブの確認方法

    3. ps -aux で現在の利用者などが表示される

    ps -u ユーザー名  で特定ユーザーのジョブだけ表示できる

    ps | grep “プロセス名”   "プロセス名"のところにsdeviceとか入れれば特定のプロセスだけを抽出できる

    top   psとは違う感じでジョブを確認できる。常に更新されるので、"q"を押すとquitできる。


  3. ジョブのkill方法

    4. kill PID(任意のPID)

    kill -KILL PID  (強制終了させる)

    killall -u “username”   で特定ユーザーを強制ログアウトできる模様。そのユーザーのジョブを全て止めてしまうため、あまり使いたくない。

    killコマンドにはオプションが色々ある。参考


  4. ログインユーザーの確認

    5. w で現在ログイン中のユーザーを表示できる


  5. ディレクトリについて

    6. cd “ディレクトリへのpath”   でそのディレクトリに移動する。pathはカレントディレクトリからの相対パスでもrootフォルダからの絶対パスでもどちらでも良い。

    ディレクトリ名は全て打たなくても、Tabキーで予測変換を出してくれる

    cd ..   で一つ上の階層に移動

    cd   で "ユーザー" の階層まで移動

    cd /   で一番上の階層まで移動。lixuxでは一番上の階層をrootディレクトリといい、フルパスを記述するときは/から書き始める必要がある。

    pwd   でカレントディレクトリへのフルパスを表示できる

    ls   でカレントディレクトリ内のファイルを表示

    ls -a   でカレントディレクトリ内の "隠しファイルも含めて" 表示


  6. ディレクトリ・ファイルの管理

    7. touch ファイル名.拡張子   カレントディレクトリにファイルを作成できる。スペースを空けて複数のファイルを作成することもできる

    cat ファイル名.拡張子   指定したファイルの中身を確認できる

    mkdir ディレクトリ名   カレントディレクトリの直下に、新たなディレクトリ(フォルダ)を作成できる。

    mv 移動させたいファイル名orディレクトリ 移動先のディレクトリ   でファイルもしくはディレクトリを指定のディレクトリに移動できる。ここでのパスは相対パスでもフルパスでもどちらでも可。

    mv 現在のファイル名 新しいファイル名   

    mv 現在のディレクトリ名 新しいディレクトリ名   

    でカレントディレクトリの下のファイル名やディレクトリ名を変更することができる。

    cp コピーしたいファイル名 複製したファイルにつけるファイル名

    cp -r コピーしたいディレクトリ名 複製したディレクトリにつけるディレクトリ名

    で任意の名前をつけてファイル(orディレクトリ)をコピーできる。パスはカレントディレクトリの直下にあるもの、もしくは絶対パスで指定する。
    ちなみに -r をつけずにディレクトリをコピーしようとするとエラーを吐く

    rm ファイル名

    rm -r ディレクトリ名

    で指定したファイルもしくはディレクトリを削除できる。多分絶対パスでもいけるけど、怖いので消したいファイルの上の階層まで移動してから使うことにする。
    cpコマンドの時と同じく、 -r をつけずにディレクトリをコピーしようとするとエラーを吐く

  7. ストレージの管理

    8. df   でストレージ容量を確認できる。

    df -BG /ディレクトリ   のように入力すれば指定したディレクトリをG(ギガバイト)表示で確認できる


  8. ディスクのコピー

    9. dd "オプション"   マシンに複数のディスクがある時に、それらの間でデータをコピーできる。

    使用例：Windowsのデータが入ったディスクをlinuxで立ち上げ、Windowsのセキュリティシステムを回避して全コピする。
    ディスクのコピー元・先の名前(sdaとかsdbとか)を間違えると大変なことになるので注意。
    緊急時に備えてboot用のLinuxが入ったUSBを用意しておくと何かと便利。
