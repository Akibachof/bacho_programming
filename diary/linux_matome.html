Linux系のターミナル使い方

個人アカウントでのsuの入り方 su→エンター→rootのパスワードを入力 →ps -aux で現在の利用者などが表示される ps -u ユーザー名 で特定ユーザーのジョブだけ表示できる ps | grep “プロセス名” ジョブ名のところにsdeviceとか入れれば特定のプロセスだけを抽出できる ジョブのkill方法 →kill PID(任意のPID) さらに強いコマンド kill -KILL PID 参考：uxmilk.jp/50638 top で作業中のジョブが出せる w で現在ログイン中のユーザーを表示できる killall -u “username” で特定のユーザーを強制ログアウトできるが、そのユーザーの全てのジョブを止めてしまうため、あまり使いたくない q でtopの表示からquitできる cd “ディレクトリ名” でそのディレクトリに移動、ディレクトリ名は全て打たなくてもTabで予測変換を出してくれる pwd で現在のディレクトリの場所を表示 cd .. で一つ上の階層に移動 cd でユーザーの階層まで戻る ls でディレクトリ内のファイルを表示 touch ファイル名.拡張子 現在のディレクトリにファイルを作成できる。スペースを空けて複数のファイルを作成することもできる cat ファイル名.拡張子 指定したファイルの中身を確認できる mkdir ディレクトリ名 新たなディレクトリ(フォルダ)を作成できる linux系で一番上のディレクトリはルートディレクトリと呼ばれ、ディレクトリを移動するときもフルパスは/から始める必要がある(はず) mv 移動させたいファイル名orディレクトリ 移動先のディレクトリ でファイルもしくはディレクトリを指定のディレクトリに移動できる mv 現在のファイル名 新しいファイル名 mv 現在のディレクトリ名 新しいディレクトリ名 でファイル名やディレクトリ名を変更することもできる cp コピーしたいファイル名 複製したファイルにつけるファイル名 任意の名前をつけてファイルをコピーできる cp -r コピーしたいディレクトリ名 複製したディレクトリにつけるディレクトリ名 任意の名前をつけてディレクトリをコピーできる。-r をつけずにディレクトリをコピーしようとするとエラーを吐く rm ファイル名 rm -r ディレクトリ名 で指定したファイルもしくはディレクトリを削除できる df でストレージ容量を確認できる。 df -BG /ディレクトリ のように入力すれば指定したディレクトリをG(ギガバイト)表示で確認できる dd 複数のディスクがあるときに、それらの間でデータをコピーできる。Windowsのデータが入ったディスクをlinuxで立ち上げ、Windowsのクソセキュリティシステムを回避して全コピできる古典的な手法。ディスクのコピー元・先の名前(sdaとかsdbとか)を間違えると大変なことになるので注意。緊急時に備えてboot用のLinuxが入ったUSBを用意しておくと何かと便利。
